2018.1.21——Golang小知识

看项目代码看见的关于string与slice之前的转化

首先上我看到的代码

func (this *YamlConfig) MustValue(key string, defaultVal ...string) string {
	val, err := this.Value(key)
	if len(defaultVal) > 0 && err != nil {
		return defaultVal[0]
	}
	return val
}

对于defaultVal是一个string类型的变量，可是在return后面返回的是defaultVal[0]，，，这是干什么呢？我很疑惑。。

后来我尝试输出一个string类型变量的酱紫形式:

package main

import "fmt"

func main(){
  str:="23"
  fmt.Println(str[0])
}

得到的结果是:50

So,可以知道，它输出的是2的ascii码值

哦，那上面我有疑惑的代码返回 的是一个ascii值

欸，不对啊，我仔细一看，返回值类型是string啊

ASCII是int型的变量啊= =

那么我对于上述有疑问的代码我就更加不解了

后来我又做了几次尝试

如下:

package main

import (
	"fmt"
)

func func_name(str ...string) string {
	return str[0]
}

func main() {
	var str string
	str = "23"
	/*str := []string{"2", "3"}*/
	fmt.Println(str[0]) //2的ascii码
	fmt.Println(str)
	fmt.Println(func_name(str))
}

这样的输出是:50

​ 23

​ 23

我又发现了defaultVal …string前面的三个‘.’

那又是什么呢？

我把前面的测试代码中的str换成注释里面的slice类型之后会报错，大致意思是应该给函数func_name()传string类型的参数而不应是slice类型的，故我可以初步认为那三个’.‘是把string变成slice类型了，那么str[0]应该是2呀，怎么是23呢?后来我又尝试输出了一下str[1]，编译器报错提示超范围了，啊，，原来是酱紫，我豁然开朗，三个‘.’不仅把string类型变成slice类型而且把整个string全部存在了第一个索引上！

接下来我对于我有疑惑的代码就可以合理的解释啦

如何解释自己思考咯，，，
